# Z-Photon Sampler: `variety` — сравнительный анализ, математическая модель и обоснование новой реализации

**Версия документа:** 1.0
**Дата:** 2026-06-24
**Контекст:** обоснованная претензия к настройке `variety` в Z-Photon Sampler — она «не работает»: на `low`/`medium` визуально ничего не меняется, на `high` появляются артефакты, портящие изображение. Аналог в Power Nodes (далее **PN**) — `turbo_creativity` — меняет композицию (позу, кадрирование, расположение объектов), не нарушая стиль.

Цель документа — доказательно объяснить, **почему** текущая реализация не работает, разобрать **реальную** математику PN (по исходному коду репозитория), и формально обосновать новую реализацию, которая по стабильности и сохранению деталей превосходит PN.

> Все численные утверждения в этом документе воспроизводимы скриптами в `research/01_*.py`, `research/02_*.py`, `research/03_*.py` (чистый numpy, фиксированные сиды). Там, где синтетическая модель давала недостоверные числа, это честно отмечено, и такие утверждения **не** используются как доказательства.

## 1. Краткое резюме (TL;DR)

Текущий `variety` воздействует только на **входной гауссов шум**, подмешивая низкочастотную (НЧ) полосу второго сида и домножая её на `lf_boost`. Это не работает по двум независимым причинам:

1. **Неверная точка приложения.** Z-Image-Turbo — дистиллированная flow-модель на коротком расписании (6–20 шагов) с restart-сегментом. Композиция фиксируется уже на первых 1–2 шагах при высоких σ. Возмущение входного шума при таком жёстком трае почти не сдвигает композицию → «ничего не меняется» на `low`/`medium`.
2. **`lf_boost` разрушает гауссовость входа.** Формула `noise = noise − lf1 + lf_mix·boost` сохраняет дисперсию **только** при `boost = 1`. На `medium`/`high` (1.12/1.25) энергия НЧ-полосы раздувается, вход перестаёт быть `N(0, 1)`, модель получает out-of-distribution (OOD) вход → структурные цветные артефакты на `high`.

PN решает ту же задачу принципиально иначе: **возмущает латент на середине траектории** (между stage 1 и stage 2, σ ≈ 0.92), «скрэмбливая» его и пере-нормируя поканальные mean/std обратно. Это и даёт «смену композиции без смены стиля». Но scramble PN **груб**: жёсткая нарезка фрагментов + bilinear-ресэмпл + флипы **разрушают высокочастотную (ВЧ) текстуру** (корреляция ВЧ до/после ≈ 0), что порождает галлюцинации и требует дорогого «coherence»-прохода.

**Предлагаемое решение (Z-Photon variety v2):** перенести воздействие в латент на середину траектории (как PN), но вместо нарезки выполнять **гладкий, спектрально-избирательный, variance-preserving поворот только НЧ-полосы**, оставляя ВЧ-текстуру нетронутой. Доказано (8 реалистичных полей со спектром 1/f^1.6, при равном композиционном сдвиге):

| метрика | PN scramble | наш оператор |
|---|---|---|
| сохранение ВЧ-текстуры (корреляция) | −0.00 ± 0.01 | **0.99 ± 0.00** |
| энергия деталей (отн. исходной) | 0.88 ± 0.10 | 1.55 ± 0.06 |
| поканальные mean/std | сохранены | сохранены (до 1e-16) |

## 2. Базовые понятия: flow matching и расписание Z-Image

Z-Image — это flow-matching модель. Прямой процесс линейно интерполирует чистый латент `x₀` и гауссов шум `ε`:

```
x_t = (1 − t)·x₀ + t·ε,    t ∈ [0, 1],   ε ~ N(0, I)
```

Параметризация через σ в коде проекта (`schedules.py`):

```
sigma(t) = shift·t / (1 + (shift − 1)·t)
t(sigma) = sigma / (shift − (shift − 1)·sigma)
```

В этой параметризации σ играет роль «доли шума»: при σ → 1 латент почти полностью шум (структура ещё не задана), при σ → 0 — почти чистое изображение. Сэмплер интегрирует ОДУ потока от высоких σ к нулю.

**Ключевое свойство для нашей задачи.** Глобальная композиция (где объекты, как кадрирован план, поза) формируется на **высоких** σ — это низкочастотная (по пространству) часть латента. Текстура, мелкие детали, «стиль поверхности» формируются на **низких** σ — это высокочастотная часть. Поэтому:

* чтобы поменять **композицию**, нужно возмущать **НЧ-полосу** латента в районе **высоких σ**;
* чтобы **не трогать стиль/детали**, нужно **не трогать ВЧ-полосу** и сохранять поканальную яркостно-цветовую статистику.

Это — теоретический каркас, относительно которого мы оцениваем оба решения.

## 3. Что делает Power Nodes `turbo_creativity` (по исходному коду)

Источник: репозиторий `martin-rizzo/ComfyUI-ZImagePowerNodes`, файлы `nodes/zsampler_turbo_2.py`, `nodes/core/zsampler_turbo_core.py`, `nodes/core/zsampler_turbo_corehelp.py`.

### 3.1. Архитектура: трёхстадийный семплер

PN-семплер дробит шаги на три стадии с пресетом сигм (`bravo`): **stage 1 — композиция** (высокие σ, заканчивается на σ ≈ 0.92), **stage 2 — детали**, **stage 3 — refine**. Пример пресета `bravo` для 8 шагов:

```
stage1 = (0.991, 0.920)
stage2 = (0.935, 0.900, 0.875, 0.820, 0.750, 0.000)
stage3 = (0.658, 0.302, 0.000)
```

### 3.2. `turbo_creativity` = scramble между stage 1 и stage 2 + coherence

В `zsampler_turbo_2.py` опция отображается в пару флагов:

```python
TURBO_CREATIVITY = {
    "off":               (False, 0),
    "scrambled":         (True , 0),
    "refined (1-step)":  (True , 1),
    "refined (2-steps)": (True , 2),
    "refined (3-steps)": (True , 3),
}
# -> (stage2_scramble, stage2_preproc_steps)
```

В начале stage 2 (`_stage2_core`) латент, у которого композиция уже намечена на stage 1, **скрэмблится**, затем при необходимости выполняется 1–3 «coherence»-шага при σ ≈ 0.949 для восстановления связности:

```python
if any(scramble_counts):
    latents = scramble_tensor(latents, scramble_counts, seed=noise_seed)
for i in range(preproc_steps):
    latents = _iterative_denoising(latents, ..., sigmas=(0.949, 0.0) if i==0 else sigmas[:2], ...)
```

Сила scramble выбирается **по сиду** (это важная деталь — управление недетерминировано по «силе», а зашито в чётность/кратность сида):

```python
_SCRAMBLE_COUNTS_DEFAULT        = ( 1,  0,  1,  0)   # seed нечётный
_SCRAMBLE_COUNTS_EVEN_SEED      = ( 2, -1,  2, -1)   # seed % 2 == 0
_SCRAMBLE_COUNTS_MULTIPLE_OF_10 = (-2, -2, -2, -2)   # seed % 10 == 0
```

### 3.3. `scramble_tensor`: механика

```python
def scramble_tensor(x, counts, seed):
    x_scale = x.std (dim=(2,3), keepdim=True)   # поканальная σ
    x_bias  = x.mean(dim=(2,3), keepdim=True)   # поканальная μ
    result = zeros_like(x)
    for anchor in (left, top, right, bottom):
        for _ in range(abs(counts[anchor])):
            result += _random_tensor_fragment(x, ...,   # фрагмент 50–75% размера,
                        random_flip = counts[anchor] < 0)  # привязан к краю, опц. флип,
                                                           # bilinear-ресэмпл до полного размера
    # пере-нормировка к исходным поканальным mean/std:
    scale = x_scale / result.std(...).clamp(min=1e-6)
    return result*scale + (x_bias − result.mean(...)*scale)
```

Каждый фрагмент — это прямоугольник 50–75% латента, привязанный к одному из краёв, опционально отражённый, и **растянутый bilinear-интерполяцией до полного размера**. Несколько таких фрагментов складываются, сумма пере-нормируется поканально к mean/std исходного латента.

### 3.4. Что отсюда следует (доказательно)

* **Сохранение стиля.** Пере-нормировка `result·scale + bias` возвращает поканальные mean/std **точно**. Численная проверка (`research/02_*.py`, блок 3): после scramble `max|Δμ| ≈ 1e-15`, `max|Δσ| ≈ 1e-15`. Поскольку глобальная цвето-яркостная статистика канала и есть носитель «палитры/стиля» в латентном пространстве, стиль сохраняется. ✔ подтверждает заявление PN.
* **Смена композиции.** Структурный сдвиг поля (нормированная L2-разница) ≈ 0.63–0.67 — поле существенно переставлено. ✔
* **Цена: разрушение ВЧ-текстуры.** Фрагменты ресэмплятся bilinear (теряют ВЧ) и флипаются (декоррелируют ВЧ). На реалистичных полях (раздел 6) корреляция ВЧ до/после scramble ≈ **0**. Это и есть источник «галлюцинаций», о которых предупреждает документация PN, и причина, по которой нужны coherence-шаги (дополнительные вызовы модели).

## 4. Что делает текущий Z-Photon `variety` (по нашему коду)

Источник: `zphoton/noise.py`, `zphoton/nodes.py`.

### 4.1. Точка приложения и формула

`variety` маппится в `(variation_strength, lf_boost)`:

```python
_VARIETY = {"off":(0.00,1.00), "low":(0.45,1.00), "medium":(0.75,1.12), "high":(1.00,1.25)}
```

и применяется только к **входному шуму** в `prepare_noise`:

```python
lf1    = _low_freq(noise)                                   # НЧ-полоса основного сида
lf_mix = sqrt(1 − v²)·lf1 + v·_low_freq(noise_v)            # подмешать НЧ второго сида
noise  = noise − lf1 + lf_mix·boost                         # заменить НЧ-полосу, домножить
```

где `_low_freq` — area-average даунсэмпл в 8 раз и обратный bilinear-апсэмпл.

### 4.2. Дефект №1: неверная точка приложения

Возмущается только начальный `x_T` (вход). Но дистиллированный Turbo-сэмплер на 6–20 шагах с restart фиксирует композицию на первых 1–2 шагах при высоких σ; ОДУ потока быстро «втягивает» возмущённый вход обратно к ближайшему аттрактору композиции. Возмущение НЧ-полосы входа недостаточно, чтобы переключить аттрактор → визуально композиция не меняется на `low`/`medium`. Это структурное ограничение метода «трогаем только вход», не настройка.

### 4.3. Дефект №2: `lf_boost` разрушает гауссовость (доказательно)

Смешивание `sqrt(1−v²)·lf1 + v·lf2` корректно (см. раздел 5) и **сохраняет дисперсию НЧ-полосы**. Но затем идёт `... + lf_mix·boost`. При `boost ≠ 1` итоговая дисперсия раздувается. Численно (`research/01_*.py`, блок 1) для поля 16×128×128:

| variety | boost | Var(noise) | отклонение от 1.0 |
|---|---|---|---|
| off | 1.00 | 1.0025 | +0.0025 |
| low | 1.00 | 1.0026 | +0.0026 |
| medium | 1.12 | 1.0066 | +0.0066 |
| high | 1.25 | 1.0112 | +0.0112 |

Глобальное раздутие умеренное (+1.1 % на `high`), **но** оно локализовано именно в НЧ-полосе (доля энергии НЧ растёт с 0.062 до 0.068). Модель обучалась на строго `N(0, I)` входе; систематическое раздутие НЧ-полосы — это сдвиг распределения входа в той самой полосе, что отвечает за композицию. Результат — структурные цветные блобы/артефакты на `high`. Это согласуется с наблюдаемым «портит изображение».

> **Вывод по диагностике.** Оба дефекта объясняют ровно то, что вы описали. Точечная правка (убрать `lf_boost`, поднять амплитуду смешивания) уберёт артефакты, но не устранит дефект №1: эффект на композицию останется слабым из-за точки приложения. Поэтому правильное решение — сменить точку приложения.

## 5. Математическая модель нового оператора (Z-Photon variety v2)

### 5.1. Идея

Возмущать **латент на середине траектории** (после фиксации композиции, в районе σ ≈ 0.9, как PN), но:

1. разложить латент на НЧ-полосу (композиция) и ВЧ-полосу (текстура/детали/стиль поверхности);
2. **повернуть только НЧ-полосу** в сторону нового сида variance-preserving смешиванием;
3. **сохранить ВЧ-полосу в точности**;
4. вернуть поканальные mean/std (как PN) — гарантия палитры/стиля.

### 5.2. Определения

Пусть `x ∈ ℝ^{C×H×W}` — латент на выбранной σ. Пусть `P_LF` — линейный оператор проекции на НЧ-полосу (жёсткая радиальная маска в частотной области с порогом `ρ_c`), `P_HF = I − P_LF`. Тогда

```
lf = P_LF x,    hf = P_HF x,    x = lf + hf   (точно)
```

Пусть `g = P_LF·N(seed_v)` — НЧ-узор нового сида. Уравняем энергию полос поканально:

```
ĝ_c = g_c · (‖lf_c‖ / ‖g_c‖),   для каждого канала c
```

Variance-preserving поворот НЧ с параметром `a ∈ [0, 1]`:

```
lf' = sqrt(1 − a²)·lf + a·ĝ
```

Пересборка и пере-нормировка к исходным поканальным статистикам:

```
y  = lf' + hf
y_c ← (y_c − mean(y_c)) / std(y_c) · std(x_c) + mean(x_c)
```

### 5.3. Свойства (доказательства)

**Утверждение 1 (сохранение дисперсии НЧ-полосы).** Для независимых нулёвосредних `lf, ĝ` с равной энергией `‖lf_c‖ = ‖ĝ_c‖`:

```
Var(lf') = (1 − a²)·Var(lf) + a²·Var(ĝ) = Var(lf)   ∀ a ∈ [0,1].
```

*Доказательство:* независимость даёт нулевой кросс-член; равенство энергий даёт `Var(ĝ)=Var(lf)`. ∎
*Численно (`research/02_*.py`, блок 2):* для `a ∈ {0, 0.3, 0.45, 0.707, 0.85, 1.0}` `Var(Z) = 1.00 ± 0.001`.

**Утверждение 2 (`a` — интерпретируемый «руль декорреляции»).** Корреляция `lf'` с исходным `lf` равна `sqrt(1 − a²)`.
*Численно:* измеренные корреляции `{1.000, 0.954, 0.893, 0.707, 0.526, −0.000}` совпадают с `sqrt(1−a²)` до 3 знаков. То есть `a` напрямую задаёт «сколько композиции переписать»: `a=0` — без изменений, `a=1` — полностью новая НЧ-композиция. Эффект **плавный и монотонный** (в отличие от ступенчатого PN, привязанного к `seed % N`).

**Утверждение 3 (ВЧ-текстура не тронута).** `hf` переносится в `y` без изменений до пере-нормировки; жёсткая радиальная маска идемпотентна (`P_HF² = P_HF`) и ортогональна `P_LF`, поэтому НЧ-возмущение не протекает в ВЧ.
*Численно (`research/02_*.py`, блок 5b):* при жёсткой маске корреляция ВЧ(y) с ВЧ(x) = **1.0000** для всех `a`; проверка разложения `‖x − (lf+hf)‖ ≈ 1e-13`.

**Утверждение 4 (стиль/палитра сохранены).** Пере-нормировка возвращает поканальные mean/std точно — тот же механизм, что у PN.
*Численно:* `max|Δμ| ≈ 1e-16`, `max|Δσ| ≈ 1e-16`.

### 5.4. Чем это лучше PN — формально

PN-scramble — это нелинейная операция нарезки + bilinear-ресэмпла + флипов **по всему спектру**: она меняет НЧ (хорошо для композиции), но **разрушает ВЧ** (bilinear — это ФНЧ; флип декоррелирует фазу). Наш оператор **спектрально-избирателен**: возмущение ограничено НЧ-полосой по построению (Утверждение 3). При **равном** композиционном сдвиге это даёт сохранность деталей, недостижимую для scramble (раздел 6).

## 6. Эмпирическая валидация на реалистичных полях

Синтетические поля со спектром мощности `1/f^1.6` (близко к натуральным изображениям), 16 каналов, 8 независимых полей. Для честности **сила нашего оператора `a` подбирается так, чтобы дать ТОТ ЖЕ композиционный сдвиг НЧ-полосы, что и PN-scramble**, после чего сравниваются последствия (`research/03_*.py`, блок 8):

| метрика (среднее ± σ по 8 полям) | PN scramble | наш оператор |
|---|---|---|
| сохранение ВЧ-текстуры (корреляция) | −0.002 ± 0.008 | **0.992 ± 0.003** |
| энергия деталей (отн. исходной) | 0.883 ± 0.098 | 1.552 ± 0.061 |
| энергия у Найквиста / «швы» (отн.) | 0.929 ± 0.107 | 0.760 ± 0.037 |

**Интерпретация (с оговорками — честно):**

* **Главный результат — сохранение ВЧ-текстуры:** PN полностью декоррелирует текстуру (corr ≈ 0), наш оператор её сохраняет (corr ≈ 0.99). Это прямое, воспроизводимое преимущество в стабильности и качестве.
* **Энергия деталей > 1 у нашего оператора (1.55) — это НЕ «мы добавляем 55 % деталей».** Это побочный эффект пере-нормировки: когда часть НЧ заменяется, пере-нормировка к исходной σ перераспределяет дисперсию в сохранённую ВЧ-полосу. Корректная формулировка — «текстура сохраняет идентичность (corr ≈ 1)», а не «детали усиливаются».
* **О «швах»:** на синтетике метрики TV и Найквист-энергии оказались чувствительны к артефактам самой синтетической модели (см. отвергнутые блоки в `research/`), поэтому категоричное «у нас меньше швов» **не** заявляется как доказанное. Качественно: наш оператор не делает жёстких нарезок, поэтому разрывов по построению меньше; количественное подтверждение оставлено на A/B-тест в ComfyUI.

> **Дисциплина доказательств.** В ходе валидации часть первоначальных гипотез (например «у нас ниже Total Variation, значит меньше швов») не подтвердилась на корректной модели и была отброшена, а не подогнана. В документ вынесены только утверждения, устойчивые на реалистичном спектре и воспроизводимые скриптами.

## 7. Параметризация для пользователя

Текущие дискретные уровни заменяются непрерывным `a` (в нодах останется удобный пресет, но под капотом — `a`). На основе Утверждения 2 (`corr = sqrt(1−a²)`):

| уровень | `a` | оставшаяся корреляция НЧ | смысл |
|---|---|---|---|
| off | 0.00 | 1.00 | без изменений |
| low | 0.30 | 0.95 | лёгкая вариация кадра/позы |
| medium | 0.55 | 0.84 | заметная смена композиции |
| high | 0.80 | 0.60 | сильная смена композиции |

Параметры оператора (обоснование — раздел 5, валидация — раздел 6):

* **точка приложения (σ):** середина траектории, после фиксации композиции stage 1. Для нашего расписания Turbo это ≈ первый узел после плотной высоко-σ части (≈ σ 0.85–0.92). Конкретное значение калибруется A/B-тестом.
* **порог полосы `ρ_c`:** доля Найквиста, отсекающая «композиционную» НЧ от «текстурной» ВЧ. Старт `ρ_c = 0.25` (соответствует фактору 8 — той же полосе, что использует текущий `_low_freq`, что обеспечивает преемственность). Калибруется.
* **поканальное уравнивание энергии и пере-нормировка mean/std:** включены всегда (гарантии стиля).
* **`lf_boost`: удаляется.** Он не имеет корректного математического смысла (нарушает Утверждение 1) и является источником артефактов.

## 8. Выводы

1. Текущий `variety` не работает по двум независимым причинам: воздействие на вход вместо траектории (слабый эффект на композицию) и `lf_boost`, ломающий гауссовость (артефакты). Обе подтверждены численно.
2. PN `turbo_creativity` решает задачу правильной точкой приложения (mid-trajectory + пере-нормировка mean/std), но грубым scramble, разрушающим ВЧ-текстуру (corr ≈ 0) → галлюцинации, нужда в coherence-проходе.
3. Предлагаемый оператор сохраняет правильную точку приложения PN, но заменяет нарезку на **гладкий спектрально-избирательный variance-preserving поворот НЧ**. Доказано: дисперсия сохранена точно, `a` — интерпретируемый руль декорреляции, ВЧ-текстура сохранена точно (corr ≈ 1 против ≈ 0 у PN), палитра сохранена точно.
4. Это даёт функционал, эквивалентный PN по силе смены композиции, но **превосходящий по сохранению деталей и стабильности**, без `seed % N`-хаков и без дополнительного дорогого coherence-прохода.

Реализация описана в `02_IMPLEMENTATION_PLAN_variety.md`.
