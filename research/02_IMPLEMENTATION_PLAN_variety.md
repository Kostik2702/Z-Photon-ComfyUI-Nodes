# Implementation Plan — Z-Photon Sampler `variety` v2

**Версия:** 1.0 · **Дата:** 2026-06-24 · **Основание:** `01_RESEARCH_variety.md`

Цель: заменить нерабочий `variety` (возмущение входного шума + `lf_boost`) на **mid-trajectory спектрально-избирательный variance-preserving поворот НЧ-полосы латента**. Старый семплер **не трогаем** — работаем с копией `*_dev` для A/B-сравнения.

---

* Минимум кода. Никаких спекулятивных абстракций.
* Хирургические правки: каждая изменённая строка трассируется к задаче.
* Старый код не «улучшаем» и не рефакторим.
* Success criteria формулируются как проверяемые цели, цикл до их выполнения.

## 1. Что НЕ трогаем

* `zphoton/noise.py` — текущий `prepare_noise` остаётся как есть (его использует и старый семплер, и refiner, и autoqc). `lf_boost`/`variation_strength` в нём не удаляем глобально — они нужны старому семплеру для честного A/B.
* `zphoton/sampling.py` — `run_sampling` менять можно **только аддитивно** (новый необязательный аргумент), чтобы не сломать существующих потребителей (`ZPhotonSamplerAdvanced`, `ZPhotonRefiner`, `ZPhotonAutoQC`). Старое поведение при невыставленном аргументе обязано быть бит-в-бит прежним.

## 2. Что добавляем (новые сущности)

### 2.1. `zphoton/variety.py` (новый модуль) — чистая математика оператора

Одна функция, без побочных эффектов, тестируемая отдельно:

```python
def lf_recompose(x, *, seed_v, a, cutoff=0.25):
    """
    Mid-trajectory variety: поворот ТОЛЬКО НЧ-полосы латента к новому сиду,
    variance-preserving, с сохранением ВЧ-текстуры и поканальных mean/std.

    x        : (B, C, H, W) латент на середине траектории
    seed_v   : сид нового НЧ-узора (детерминированно от пользовательского seed)
    a        : сила в [0,1]; corr с исходной композицией = sqrt(1-a^2)
    cutoff   : доля Найквиста, граница НЧ/ВЧ полос
    return   : (B, C, H, W) с той же поканальной статистикой, изменённой только НЧ
    """
```

Реализация ровно по разделу 5 research-дока:
1. `lf = P_LF x`, `hf = x − lf` через rFFT + жёсткая радиальная маска (idempotent).
2. `g = P_LF(N(seed_v))`; поканальное уравнивание энергии `ĝ_c = g_c·‖lf_c‖/‖g_c‖`.
3. `lf' = sqrt(1−a²)·lf + a·ĝ`.
4. `y = lf' + hf`; пере-нормировка поканальных mean/std к `x`.

Замечания по реализации:
* FFT на латенте дёшев (латент ~ H/8×W/8, 16 каналов) — выполняется на устройстве `x`.
* Жёсткая маска кэшируется по `(H, W, cutoff, device)`.
* `a = 0` → ранний выход, возвращаем `x` без изменений (нулевая стоимость для `off`).
* Работа в float32 на устройстве латента; результат приводится к dtype `x`.

### 2.2. Точка вызова — внутри `run_sampling` (аддитивный аргумент)

Наш `run_sampling` уже умеет резать расписание на сегменты по граничным σ (механизм `split_idxs` для `composition_model`/`neg_cfg`, где сегменты делят общие узлы и rescale ComfyUI взаимно сокращается). Variety встраивается ровно туда же: добавляем границу `variety_end` (σ, ниже которой композиция уже зафиксирована) и **между сегментами** один раз применяем `lf_recompose` к латенту.

Новые необязательные параметры `run_sampling`:
```python
variety_a: float = 0.0,        # 0 => полностью выключено, старый путь
variety_seed: int = 0,
variety_end: float = 0.90,     # σ-граница применения (калибруется)
variety_cutoff: float = 0.25,
```
Поведение:
* если `variety_a <= 0` → ни одной новой ветки не исполняется, путь идентичен текущему (требование §1).
* иначе: добавляем индекс границы в `split_idxs`; в цикле по сегментам, при переходе через `variety_end`, применяем `lf_recompose(cur["samples"], ...)` перед запуском следующего сегмента.

> Это переиспользует уже существующий, проверенный механизм сегментации (общие узлы → точное сокращение noise_scaling). Никакой новой логики ресэмплинга не вводится.

### 2.3. Dev-ноды (копии, не override)

В `zphoton/nodes.py` **не меняем** существующие классы. Добавляем рядом:

* `ZPhotonSamplerDev` — копия `ZPhotonSampler` с:
  * `_VARIETY_DEV = {"off":0.00, "low":0.30, "medium":0.55, "high":0.80}` (значения `a`, обоснование — раздел 7 research-дока);
  * вызовом `run_sampling(..., variety_a=a, variety_seed=vseed, variety_end=..., variety_cutoff=...)` вместо старого `prepare_noise(variation_strength=..., lf_boost=...)`;
  * для variety-пути `prepare_noise` вызывается **без** `variation_strength`/`lf_boost` (чистый init-шум) — возмущение теперь в латенте.
* Регистрация в `NODE_CLASS_MAPPINGS`/`NODE_DISPLAY_NAME_MAPPINGS`: `"ZPhotonSamplerDev": "ZPhoton Sampler (dev)"`.

Старый `ZPhotonSampler` остаётся доступным в графе → прямое A/B в одном workflow.

## 4. Риски и смягчение

* **σ-граница не совпадает с узлом расписания.** Берём ближайший существующий узел ниже `variety_end` (как уже делает `first_idx_below`) — границы сегментов остаются на реальных узлах, rescale сокращается точно.
* **batch > 1.** Оператор поканальный и побатчевый (mean/std по dims=(2,3)); сид variety детерминированно сдвигается по индексу в батче, чтобы кадры в батче отличались.
* **Очень маленький латент** (H или W < 1/cutoff). Маска вырождается → `lf_recompose` возвращает `x` (ранний выход), как `_low_freq` уже делает для малых полей.
* **dtype/устройство.** Расчёт в float32 на устройстве `x`, возврат в исходный dtype.
